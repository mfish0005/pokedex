Initial vs Derived Variables

üé®

Initial Variables

Literal values that serve as the foundation. These are the raw colors, sizes, and other basic values.

$green: #50a000;
$size-1: 2.5rem;
$weight-bold: 700;
üîó

Derived Variables

Variables that inherit from initial variables for better modularity and theming.

$primary: $green;
$family-primary: $family-sans-serif;
üéØ

Best Practice

Always use derived variables when possible. This allows changing themes with minimal code changes.

Use: $primary instead of $green
Use: $family-primary instead of $family-sans-serif

Variable Hierarchy

Usage Guidelines

‚úÖ Use derived variables first

// Good - uses derived variable
.my-button {{ '{' }}
  background: $primary;
  color: $text;
{{ '}' }}

‚ùå Avoid using initial variables where possible

// Bad - makes it more work to change themes later
.my-button {{ '{' }}
  background: $green; // Use $primary instead
{{ '}' }}

All Variables

Base Colors

$white Base white color (#ffffff)
$black Base black color (#000000)
$gray-lightest Lightest gray (#fafafa) - very subtle backgrounds
$gray-lighter Lighter gray (#f5f5f5) - card backgrounds, subtle borders
$gray-light Light gray (#eeeeee) - borders, dividers, inactive states
$gray Medium gray (#e0e0e0) - secondary text, muted elements
$gray-dark Dark gray (#bdbdbd) - icon colors, secondary borders
$gray-darker Darker gray (#9e9e9e) - placeholder text, disabled states
$gray-darkest Darkest gray (#757575) - primary text, strong elements

Color Palette

$green / $green-light / $green-dark Green color variants for primary/success states
$blue / $blue-light / $blue-dark Blue color variants for info/primary states
$red / $red-light / $red-dark Red color variants for danger/error states
$orange / $orange-light / $orange-dark Orange color variants for warning states
$yellow / $yellow-light / $yellow-dark Yellow color variants for warning/attention
$purple / $purple-light / $purple-dark Purple color variants for accent states

Semantic Colors

$primary Main brand color (buttons, links, focus states) - derived from $green
$secondary Secondary color (outline buttons, muted elements) - derived from $gray-dark
$success Success color (positive feedback, confirmations) - derived from $green
$warning Warning color (cautions, alerts) - derived from $orange
$danger Danger color (errors, destructive actions) - derived from $red
$info Info color (neutral information, tips) - derived from $blue
$text Default text color for readability - derived from $gray-darker

Typography

$family-monospace Monospace font family (code, technical content) - initial variable
$family-sans-serif Sans-serif font family (UI, headings, body text) - initial variable
$family-primary Main font family (headings, UI elements) - derived from $family-sans-serif
$family-secondary Secondary font family (code blocks, accents) - derived from $family-monospace

Font Sizes

$size-1 Largest size (2.5rem - hero titles) - initial variable
$size-2 Very large (2rem - page titles) - initial variable
$size-3 Large (1.75rem - section headings) - initial variable
$size-4 Medium-large (1.5rem - subsections) - initial variable
$size-5 Medium (1.25rem - large text) - initial variable
$size-6 Base size (1rem - body text) - initial variable
$size-7 Small (0.75rem - captions, small text) - initial variable
$sizes Array of all sizes for helper class generation - derived variable

Font Weights

$weight-light Light weight (400) for subtle text - initial variable
$weight-normal Normal weight (500) for body text - initial variable
$weight-semibold Semibold weight (600) for emphasis - initial variable
$weight-bold Bold weight (700) for strong emphasis - initial variable

Color Maps

$colorMap Map of color names to values (used by .has-text-* and .has-background-* classes) - derived variable
$shadeMap Map of shade names to values (used for hover states and variations) - derived variable
